What tools are available?

Lin Clark, a very active Drupal contributor, is currently preparing a module that enables microdata and RDF to happily co-exist. St√©phane Corlosquet created a new Drupal module for mapping Drupal content structure to schema.org vocabularies.

The software library and service any23, which also happens to power the data acquisition module of the semantic indexer Sindice will very soon release support for microdata based on the mapping defined, here.

OpenLink's Virtuoso Sponger supports Microdata natively and turns it into Linked Data. Further, OpenLink's URIBurner (a data virtualization service that transforms data hosted in a variety of data spaces and formats into standards compliant Linked Data Objects for uniform access, integration and management) supports microdata amongst many other input formats.

The upcoming release of TopBraid Composer will include comprehensive support for editing and processing Schema.org Microdata.

Ed Summers is developing an rdflib plugin that enables to parse Microdata and process the resulting RDF.

Michael has released the omnidator (the omnipotent data translator), an online tool and (CORS-enabled) API to translate formats that use Schema.org terms into RDF. Currently, microdata (using Ed Summers' rdflib-microdata plugin) and CSV as input formats are supported.
